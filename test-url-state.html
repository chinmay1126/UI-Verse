Shareable URL State for Every Filter

📖 API

Usage Reference

Set Search

URLState
  .onSearchChange(
    'button'
  );

Set Category

URLState
  .onCategoryChange(
    'cards'
  );

Set Sort

URLState
  .onSortChange(
    'newest'
  );

Get State

const state =
  URLState
    .getState();
// { search, cat,
//   sort }

Get URL

const url =
  URLState
    .getShareableURL();
// full URL string

Clear State

URLState
  .clearState();
// resets URL
// params

Observe Changes

URLState
  .onChange(
    (state) => {
      render(state);
    }
  );

Init on Load

document
  .addEventListener(
    'DOMContent'
    + 'Loaded',
    () => URLState
      .init()
  );
🔌 Integration

Framework Examples

Vanilla JS

const input =
  document.getElementById(
    'search'
  );

input.addEventListener(
  'input',
  e => URLState
    .onSearchChange(
      e.target.value
    )
);

React Hook

useEffect(() => {
  URLState.onChange(
    (state) => {
      setFilter(
        state.category
      );
    }
  );
}, []);

Vue Watch

watch(search,
  (val) => {
    URLState
      .onSearchChange(
        val
      );
  }
);

Restore on Load

const state =
  URLState.init();

searchInput.value =
  state.searchQuery;

setCategory(
  state.category
);
🔒 Security

Built-in Safeguards

🔐

XSS Prevention

All values written to the URL are sanitised and HTML-encoded before being read back into the DOM.

📏

Length Limits

Search queries are capped at 200 characters. Category and sort params are validated against an allowlist.

🧹

Clean URLs

Default values are never written to the URL — only non-default state produces URL params.
